# コードベース分析スキル

このスキルを使って、コードベース全体を体系的に分析し、改善点をレポートします。

## 実行手順

### 1. プロジェクト把握（必ず最初に実行）

```bash
find . -type f -not -path './.git/*' -not -path './node_modules/*' -not -path './.next/*' | sort
```

上記の結果をもとに：
- 技術スタック（言語、フレームワーク、ツール）を特定
- プロジェクト規模（ファイル数・ディレクトリ構成）を把握
- 主要な設定ファイル（package.json, tsconfig.json, etc.）を読む

### 2. 多角的分析の実施

以下の観点でコードを読み込み、問題点を収集する：

**🏗️ アーキテクチャ**
- レイヤー分離と責任の明確さ
- モジュール間の依存関係（循環依存）
- API設計の一貫性

**🔧 コード品質**
- 重複ロジックの存在
- 関数・クラスの複雑度（大きすぎる関数）
- 命名の適切さと一貫性
- エラーハンドリングの網羅性

**🔒 セキュリティ**
- ハードコードされた認証情報
- 入力バリデーション・サニタイズ
- 認証・認可の抜け
- 依存関係の脆弱性リスク

**⚡ パフォーマンス**
- N+1クエリ・不要なループ
- キャッシュ活用の余地
- 非同期処理の最適化
- リソースリーク

**🧪 テスト**
- テストが存在しない重要なロジック
- テストの質（エッジケースのカバー）

**📦 依存関係・保守性**
- 古い・非推奨のライブラリ
- 設定のハードコード
- ドキュメント整備状況

### 3. レポート生成

以下の形式でレポートをまとめる：

```
## コードベース分析レポート

### プロジェクト概要
[技術スタック、規模、主要コンポーネントをまとめる]

### 🔴 CRITICAL（即対応）
[セキュリティリスク・重大バグ等]

### 🟡 WARNING（優先対応）
[パフォーマンス問題・潜在的バグ・技術的負債]

### 🔵 SUGGESTION（改善推奨）
[コード品質・可読性・保守性の向上]

### ✅ 良い点
[現在の実装で評価できる点]

### 🎯 優先アクションリスト
[インパクト/コストを考慮した優先度順のアクションプラン]

### 📊 サマリーテーブル
| カテゴリ | Critical | Warning | Suggestion |
|---------|----------|---------|------------|
| セキュリティ | | | |
| パフォーマンス | | | |
| コード品質 | | | |
| テスト | | | |
| アーキテクチャ | | | |
```

## 注意事項

- 問題には必ず**ファイルパス:行番号**を明示する
- 改善策は具体的なコード例や手順を示す
- 問題点だけでなく良い実装も積極的に評価する
- プロジェクトの文脈（段階、チーム規模）を考慮した現実的な提案をする
- 一部だけ見て早計な結論を出さず、全体を確認してから判断する
